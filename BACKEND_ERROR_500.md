# Исправление ошибки 500 при удалении уведомлений

## Проблема
Backend возвращает **500 Internal Server Error** при PATCH запросе с `is_deleted: true`.

## Что сделано

1. **Добавлена обработка ошибок** в `update_notification` endpoint
2. **Добавлено логирование** для диагностики проблемы
3. **Добавлена проверка полей** перед возвратом ответа

## Что нужно сделать

### 1. ПЕРЕЗАПУСТИТЕ BACKEND! ⚠️

```powershell
# Остановите backend (Ctrl+C)
# Затем запустите снова:
cd backend
uvicorn app.main:app --reload
```

### 2. Проверьте логи backend

После перезапуска попробуйте удалить уведомление и **посмотрите логи в терминале где запущен backend**.

Там должна быть реальная ошибка, например:
```
[ERROR] Error updating notification ...: ...
Traceback (most recent call last):
  ...
```

### 3. Возможные причины 500 ошибки

1. **Проблема с сериализацией** - `NotificationRead` не может сериализовать объект
2. **Проблема с БД** - ошибка при коммите изменений
3. **Проблема с полями** - отсутствие обязательных полей

### 4. Если ошибка все еще есть

**Пришлите логи из терминала backend** - там будет реальная причина ошибки.

## Проверка

После перезапуска backend:
1. Обновите страницу (F5)
2. Попробуйте удалить уведомление
3. Проверьте логи backend в терминале
4. Проверьте Network tab - должен быть 200 OK вместо 500
