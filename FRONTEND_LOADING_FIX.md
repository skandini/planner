# Исправление: Скрытие индикатора "загружаю события" при фоновом обновлении

## Проблема

В календаре постоянно показывалась надпись "загружаю события", что создавало ощущение постоянного обновления страницы.

## Причина

События загружались каждые 12 секунд через polling, и при каждой загрузке показывался индикатор загрузки (`eventsLoading = true`).

## Решение

Добавлен параметр `silent` в функцию `loadEvents`, который позволяет загружать события в фоне без показа индикатора загрузки.

### Изменения:

1. **Добавлен параметр `silent` в `loadEvents`:**
   ```typescript
   const loadEvents = useCallback(async (silent: boolean = false) => {
     // При silent = true не устанавливаем индикатор загрузки
     if (!silent) {
       setEventsLoading(true);
     }
     // ... загрузка событий
     // В finally сбрасываем только если был установлен
     if (!silent) {
       setEventsLoading(false);
     }
   }, [...]);
   ```

2. **Фоновый polling без индикатора:**
   - Polling теперь каждые 30 секунд (вместо 12)
   - Все вызовы `loadEvents` в polling используют `silent = true`
   - Индикатор загрузки не показывается

3. **Явная загрузка с индикатором:**
   - При изменении даты/диапазона/календаря - показывается индикатор
   - После создания/обновления события - показывается индикатор
   - При первой загрузке - показывается индикатор

## Результат

- ✅ Индикатор "загружаю события" больше не мигает постоянно
- ✅ События обновляются в фоне каждые 30 секунд
- ✅ Индикатор показывается только при явных действиях пользователя
- ✅ Улучшен UX - нет отвлекающих индикаторов
