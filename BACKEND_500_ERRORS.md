# Исправление ошибок 500 на backend

## Проблема

Backend возвращает **500 Internal Server Error** для запросов:
- `GET /api/v1/calendars/{id}/members` 
- `GET /api/v1/events/?calendar_id=...&from=...&to=...`

Это вызывает CORS ошибки, потому что при 500 ошибке backend не отправляет CORS headers.

## Решение

### 1. ПЕРЕЗАПУСТИТЕ BACKEND! ⚠️

**Это самое важное!** Backend должен быть перезапущен после всех изменений:

```powershell
# Остановите backend (Ctrl+C в терминале где он запущен)
# Затем запустите снова:
cd backend
uvicorn app.main:app --reload
```

### 2. Проверьте логи backend

После перезапуска посмотрите логи в терминале где запущен backend. Там должна быть реальная ошибка, например:

```
ERROR:    Exception in ASGI application
Traceback (most recent call last):
  ...
```

### 3. Возможные причины 500 ошибок

1. **Backend не перезапущен** - самая частая причина
2. **Ошибка в коде** - проверьте логи backend
3. **Проблема с БД** - возможно, миграция не применена
4. **Проблема с импортами** - возможно, что-то не импортировано

### 4. Проверка миграций

Убедитесь, что все миграции применены:

```powershell
cd backend
python -m alembic current
```

Должно показать: `a1b2c3d4e5f6 (head)`

### 5. Если ошибка все еще есть

**Пришлите логи из терминала backend** - там будет реальная причина ошибки.

## Примечание

404 ошибка для события - это нормально, если событие было удалено. Это не проблема, просто событие больше не существует.
